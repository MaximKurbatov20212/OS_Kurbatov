Почему pwd не печатает /usr/bin?
Потому что, когда мы вызываем pwd, то создаем процесс, в который скопируется
текущая директория родительского процесса. А значит, у этого подпроцесса
будет та же рабочая директория, что и у родительского. А значит /usr/bin 
и не должен был напечататься.(Это говорит о том, что мои слова о том, что изначально
текущая рабочая директория равна той, в который находится исполняемый файл - ошибка)

Как написать программу по выводу текущей директории?

#include <stdio.h>
#include <unistd.h>

#define ERROR 1;

int main(int argc, char** argv) {
    execvp("pwd", argv);
    perror("Execvp error");
    return ERROR;
}

Как изменить рабочую директорию?
Системные вызовы chdir(2) и fchdir(2) изменяют текущую рабочую директорию вашего
процесса.

int chdir(const char *path);
path Путевое имя файла-директории, который должен стать новой текущей рабочей
директорией.

int fchdir(int fd);
fildes Файловый дескриптор открытого файла.

Возвращаемые значение:
Успех - 0
Неудача - (-1) и errno установленна

Чтобы назначить директорию текущей, процесс должен иметь право записи в эту
директорию и все её родительские директории.
